Copyright 2026 Abir Maheshwari

## 1. Incident Classification
| Severity | Description |
|----------|-------------|
| Critical | Data breach, malicious code injection, credential leak |
| High | Unpatched security vulnerability, backend outage |
| Medium | Bug in security feature, failed attestation |
| Low | Documentation error, minor UI issue |

## 2. Response Workflow
1. **Detection**: Monitor GitHub issues, user reports, and security scans
2. **Containment**: Disable affected features, revoke compromised credentials, isolate affected systems
3. **Eradication**: Patch vulnerability, remove malicious code, update dependencies
4. **Recovery**: Deploy patched release, notify users via GitHub releases
5. **Post-Mortem**: Document root cause, update threat model and security policies
