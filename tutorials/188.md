# Tutorial 188: Quantum Containers

## Objective
Package and deploy quantum applications using containerization for consistent quantum computing environments.

## Prerequisites
- Tutorials 1-187 completed
- AbirQu installed (`pip install abirqu`)

## What You'll Learn
- Quantum container architecture
- Containerized quantum workloads
- Quantum container orchestration

## Step-by-Step Code

```python
import math
import numpy as np
from abirqu import Circuit
from abirqu.primitives import QuantumRun

# ============================================
# Part 1: Quantum Container Architecture
# ============================================

print("Quantum Containers:")
print("=" * 50)

print("Container Registry:")
containers = [
    ("abirqu-base", "2.0", "50MB", "Base quantum runtime"),
    ("abirqu-ml", "1.5", "120MB", "Quantum ML libraries"),
    ("abirqu-chemistry", "1.2", "95MB", "Quantum chemistry tools"),
    ("abirqu-optimization", "1.8", "80MB", "Optimization algorithms"),
    ("abirqu-simulation", "2.1", "110MB", "Simulation frameworks"),
]

for name, version, size, description in containers:
    print(f"  {name:25s} v{version} | {size:6s} | {description}")

circuit = Circuit(2, name="ContainerBase")
circuit.h(0)
circuit.cnot(0, 1)
circuit.measure_all()

result = QuantumRun(circuit, shots=200)
print(f"\nBase container test: {result.counts}")

# ============================================
# Part 2: Containerized Quantum Workloads
# ============================================

print("\n\nContainerized Quantum Workloads:")
print("-" * 50)

workloads = [
    ("Workload-A", "abirqu-base", "2-qubit", "Active", "10ms"),
    ("Workload-B", "abirqu-ml", "4-qubit", "Active", "50ms"),
    ("Workload-C", "abirqu-chemistry", "6-qubit", "Pending", "100ms"),
    ("Workload-D", "abirqu-optimization", "8-qubit", "Active", "80ms"),
]

for workload, image, resources, status, runtime in workloads:
    print(f"  {workload:15s} | Image: {image:20s} | {resources:8s} | {status:8s} | {runtime}")

# Simulate workload execution
circuit = Circuit(4, name="ContainerWorkload")
circuit.h(0)
circuit.h(1)
circuit.cnot(0, 1)
circuit.cnot(2, 3)
circuit.measure_all()

result = QuantumRun(circuit, shots=300)
print(f"\nWorkload execution: {result.counts}")

# ============================================
# Part 3: Quantum Container Orchestration
# ============================================

print("\n\nQuantum Container Orchestration:")
print("-" * 50)

cluster_info = {
    "quantum-cluster-1": {
        "nodes": 4,
        "total_qubits": 32,
        "containers_running": 12,
        "status": "Healthy",
    },
    "quantum-cluster-2": {
        "nodes": 2,
        "total_qubits": 16,
        "containers_running": 8,
        "status": "Healthy",
    },
}

for cluster, info in cluster_info.items():
    print(f"  {cluster}:")
    print(f"    Nodes: {info['nodes']} | Total Qubits: {info['total_qubits']}")
    print(f"    Containers: {info['containers_running']} | Status: {info['status']}")

scheduling_policies = [
    ("Round-Robin", "Distribute equally across nodes"),
    ("Bin-Packing", "Fill nodes before using new ones"),
    ("Quantum-Aware", "Schedule based on qubit requirements"),
    ("Latency-Optimized", "Minimize inter-container communication"),
]

print("\nScheduling Policies:")
for policy, description in scheduling_policies:
    print(f"  {policy:20s}: {description}")

circuit = Circuit(4, name="OrchestrationTest")
circuit.h(0)
circuit.ry(1, math.pi / 4)
circuit.cnot(0, 1)
circuit.rz(2, math.pi / 3)
circuit.cnot(2, 3)
circuit.measure_all()

result = QuantumRun(circuit, shots=400)
print(f"\nOrchestration test: {result.counts}")

print("\n" + "=" * 50)
print("Quantum Containers Complete!")
print("=" * 50)
```

## Key Concepts

### Quantum Container Architecture
- Standardized quantum runtime environments
- Modular container registry for different workloads
- Lightweight, optimized container images

### Containerized Workloads
- Isolated quantum workload execution
- Resource allocation per container
- Status monitoring and lifecycle management

### Container Orchestration
- Multi-node quantum cluster management
- Intelligent scheduling policies
- Health monitoring and auto-recovery
